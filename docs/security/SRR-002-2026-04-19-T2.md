# SRR-002 - Tier 2 Security Review

- **Date**: 2026-04-19
- **Tier**: T2 (Feature Complete)
- **Scope**: Changes since SRR-001 (BUG-015/017/018, ADR-012, annotation counts, suggestions UI)

## Findings

| ID | Severity | Description | Status |
|---|---|---|---|
| S3-2 | HIGH | Auto-learn overwrites existing config keys | FIXED |
| S5-4 | MEDIUM | URL pattern matching uses String.includes() - trust gate bypass | FIXED |
| S7-3 | MEDIUM | innerHTML with hardcoded SVG in settings.js | DEFERRED (not exploitable) |
| Q3-3 | MEDIUM | F19 wrapping missing on 4 compare tools | DEFERRED (roadmap) |
| S7-4 | LOW | DOM re-append guards - assessed SAFE | NO ACTION |
| S5-5 | LOW | suggestions-ui module state not cleared on destroy | FIXED |

## Fixes Applied

### S3-2: Auto-learn config merge (HIGH → FIXED)
- **Before**: Auto-learn replaced entire config.json with `{ urlPatterns, autoAudit, smartSuggestions }`
- **After**: Reads existing config, merges auto-learned keys on top, preserves user settings
- **File**: `server/src/http-receiver.js`

### S5-4: URL hostname matching (MEDIUM → FIXED)
- **Before**: `pageUrl.includes(pattern)` matched against full URL including query params
- **After**: Parses URL, matches `hostname:port` only
- **File**: `extension/lib/sidebar/settings.js`

### S5-5: Suggestions state cleanup (LOW → FIXED)
- **Before**: `_expandedState` held DOM references after sidebar destroy
- **After**: `resetSuggestions()` called from `destroy()`, clears module state
- **Files**: `extension/lib/sidebar/suggestions-ui.js`, `extension/lib/annotation-sidebar.js`

## Deferred

### S7-3: innerHTML in settings.js
Two `innerHTML` assignments with hardcoded SVG icon strings. Not exploitable (no server/user data flows into these strings). Deferred to next refactor pass.

### Q3-3: F19 wrapping gaps
`compare-captures.js` and `compare-baseline.js` return `n.text` without `wrapCapturedText()`. `get-capture.js` and `get-latest.js` return full JSON unwrapped. Added to roadmap.
