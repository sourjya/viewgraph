# SRR-001: Security Review Report - 2026-04-18

**Reviewer:** Automated + manual review
**Scope:** Full codebase (server/src, extension/lib, extension/entrypoints, scripts)
**Previous review:** docs/security/security-assessment.md (2026-04-08)
**Changes since last review:** Server lifecycle, F19 prompt injection defense, F17 trust indicator, F18 agent guidance, M16 sidebar polish, fuzzy filename matching

---

## Executive Summary

No CRITICAL findings. 2 HIGH findings (both related to the known ADR-010 auth removal). 5 MEDIUM findings. 10 LOW/INFO findings. The codebase has strong fundamentals: consistent path validation, Zod schemas on all MCP inputs, no eval/Function, no ReDoS, robust parser. The primary risk surface remains the unauthenticated localhost HTTP receiver, which is a documented accepted risk pending native messaging (M17c).

---

## Findings

### HIGH

#### S1-1: PUT /config unauthenticated arbitrary config write
- **Confidence:** CONFIRMED
- **File:** `server/src/http-receiver.js:155-165`
- **Regression test:** None
- **Attack:** Any website the developer visits can `fetch('http://127.0.0.1:9876/config', { method: 'PUT', body: '{"urlPatterns":["*"],"trustedPatterns":["*"]}' })` to overwrite project config, bypassing F17 trust gate.
- **Remediation:** Validate config body against a schema whitelist of known keys (`urlPatterns`, `trustedPatterns`, `autoAudit`, `smartSuggestions`). Reject unknown keys. Effort: 30 min. Risk: None.

#### S1-2: POST /captures unauthenticated capture injection
- **Confidence:** CONFIRMED
- **File:** `server/src/http-receiver.js:207-260`
- **Regression test:** Format validation exists, content poisoning test missing
- **Attack:** Any website can POST a crafted capture with prompt injection payloads in annotations. The capture is indistinguishable from a legitimate one. F19 wrapping helps at the MCP layer but the capture itself is accepted.
- **Remediation:** Known accepted risk per ADR-010. Native messaging (M17c) is the fix. Interim: add `_source` metadata field that only the real extension sets.

### MEDIUM

#### S3-1: Auto-learn config writes URL pattern from untrusted capture metadata
- **Confidence:** CONFIRMED
- **File:** `server/src/http-receiver.js:241-248`
- **Regression test:** None
- **Attack:** POST to `/captures` with `metadata.url: "http://evil.com"` on a fresh project writes `urlPatterns: ["evil.com"]` to config.
- **Remediation:** Only auto-learn from localhost/127.0.0.1/file:// URLs. Effort: 15 min.

#### S5-1: No security response headers
- **Confidence:** CONFIRMED
- **File:** `server/src/http-receiver.js:74-78`
- **Remediation:** Add `X-Content-Type-Options: nosniff` and `Cache-Control: no-store` to the `json()` helper. Effort: 5 min.

#### S5-2: GET /info exposes filesystem paths to any origin
- **Confidence:** CONFIRMED
- **File:** `server/src/http-receiver.js:117-130`
- **Attack:** Any website can learn the developer's project path, captures dir, URL patterns.
- **Remediation:** Known accepted risk. Consider splitting into minimal `/info` and full `/info/details`.

#### S7-1: Shadow DOM uses mode: 'open'
- **Confidence:** CONFIRMED
- **File:** `extension/lib/annotation-sidebar.js:252`
- **Attack:** Host page JS can read annotation comments and server URL from the shadow root.
- **Remediation:** Change to `mode: 'closed'`. All internal code uses module-scoped reference. Effort: 10 min.

#### S7-2: innerHTML with server-derived version string
- **Confidence:** CONFIRMED
- **File:** `extension/lib/annotation-sidebar.js:109`
- **Attack:** Rogue localhost process serves malicious `serverVersion` in `/info` response.
- **Remediation:** Replace innerHTML with textContent + createElement. Effort: 5 min.

### LOW

#### WS-1: WebSocket has no maxPayload limit
- **File:** `server/src/ws-server.js:46`
- **Remediation:** Add `maxPayload: 1 * 1024 * 1024` to WebSocketServer constructor.

#### WS-2: WebSocket has no connection limit
- **File:** `server/src/ws-server.js:57`
- **Remediation:** Add `if (clients.size >= 10) { ws.close(1013); return; }`.

#### S5-3: Error messages may leak internal paths
- **File:** `server/src/http-receiver.js:345`
- **Remediation:** Return generic error, log details to stderr.

#### S3-2: compare-screenshots reads from parent of capturesDir
- **File:** `server/src/tools/compare-screenshots.js:41`
- **Remediation:** Use dedicated screenshots directory with validateCapturePath.

#### Q3-1: F19 wrapping gaps in 12 tools
- **File:** find-missing-testids, detect-recurring-issues, analyze-patterns, generate-spec, diff-annotations, compare-captures, compare-baseline, get-latest, get-session, check-annotation-status, check-consistency, get-capture
- **Remediation:** Add wrapCapturedText/wrapComment to all tools returning user text. See codebase review Phase 2d.

### INFO

- All 126 JSON.parse calls are guarded (try-catch or safeParse wrapper)
- All 37 MCP tools have Zod schema validation
- Zero eval/Function/new RegExp with user input
- Zero ReDoS-vulnerable patterns
- Native messaging buffer handling is correct with bounds checks
- validateCapturePath consistently used across all file-reading tools

## Positive Security Patterns

1. `validateCapturePath()` used consistently across all 37 tools
2. Zod validation on every MCP tool input
3. `safeParse()` pattern in ViewGraph v2 parser - never throws
4. No eval, no Function constructor, no dynamic regex
5. 127.0.0.1 binding - not network-accessible
6. Payload size limits: 5MB captures, 10MB snapshots
7. F19 traverser sanitization: comment stripping, data-* capping, hidden text clearing
8. AbortSignal.timeout on all extension fetch calls
9. No cookie/localStorage/sessionStorage access from content script
10. All HTTP POST body parsing wrapped in try-catch
